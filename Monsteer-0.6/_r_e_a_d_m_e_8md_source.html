4 # Overview
5 
6 Monsteer is a library for Interactive Supercomputing in the neuroscience
7 domain. Monsteer facilitates the coupling of running simulations
8 (currently NEST) with interactive visualization and analysis
9 applications. Monsteer supports streaming of simulation data to clients
10 (currenty only spikes) as well as control of the simulator from the
11 clients (also kown as computational steering). Monsteer's main
12 components are a C++ library, a MUSIC-based application and Python
13 helpers.
14 
15 # Features
16 
17 Monsteer provides the following functionality:
18 * A brion::SpikeReportPlugin for streaming spike data using ZeroEQ. The
19  plugin accepts URIs with the format "monsteer://[host[:port]]".
20 * A MUSIC application called music_proxy to be used as the runtime gateway
21  to simulators that support MUSIC, e.g. NEST.
22 * A small Python library to interface the Simulator in the client side and
23  MUSIC proxy on the simulator side. This library also activates the Brion
24  plugin when imported.
25 
26 # Examples
27 
28 The directory *examples/nest* contains two simple examples using NEST. For each
29 example there is Python script (the simulator code) and a MUSIC configuration
30 file. The music configuration files have instructions on how to run them. The
31 first example, (files *minimal.py* and *nest2music_proxy.music*), is a minimal
32 simulation with streaming enabled. The second example (*minimal_steered.py*
33 and *nest2music_proxy_with_steering.music*) is an extended version of the
34 first one that adds basic steering support.
35 
36 This package does not provide any client code example at the moment. As a
37 streaming client you can use the spikeConverter tool from
38 [Brion](https://github.com/BlueBrain/Brion.git) as a reference.

40 # Building from Source
41 
42 Monsteer is a cross-platform library, designed to run on any modern operating
43 system, including all Unix variants. It requires a C++11 compiler and uses CMake
44 to create a platform-specific build environment. The following platforms and
45 build environments are tested:
46 
47 * Linux: Ubuntu 16.04, RHEL 6.8 (Makefile, Ninja)
48 * Mac OS X: 10.9 (Makefile, Ninja)
49 
50 Building from source is as simple as:
51 
52  git clone https://github.com/BlueBrain/Monsteer.git
53  mkdir Monsteer/build
54  cd Monsteer/build
55  cmake -GNinja ..
56  ninja
