1 VSD computation {#computeVSD}
2 ===============
3 
4 The compute-vsd application was developed with the goal of helping scientists
5 to do their experiments and research in the context of in-silico VSD imaging.
6 The design and validation process was done in collaboration with them.
7 
8 It uses the existing Fivox library, with a few modifications and additions, such
9 as a new loader (VSDLoader).
10 
11 ## Usage
12 
13  compute-vsd --volume 'fivoxvsd://BlueConfig?report=voltages&target=Column&areas=/path/to/report/area.bbp&dt=0.1'
14  --curve /path/to/dyecurve.txt --depth 2000 [--interpolate-attenuation]
15  --v0 -65 --g0 10000 [--ap-threshold -55] [--export-volume] [--export-point-sprites]
16  --sigma 0.001 --sensor-res 512 --sensor-dim 1000
17  --frame[s] '0 10' (or alternatively, --time[s] '0 100') --output /path/to/output/prefix_
18 
19 ### Parameters:
20 
21 ##### Volume (fivoxvsd://)
22 
23 * BlueConfig: Path to the BlueConfig file
24 * report: Name of the voltage report in the BlueConfig
25 * target: Name of the Cell Target
26 * areas: Path to the area report file
27 * dt: Timestep between requested frames in milliseconds
28 * gidFraction: Take random cells from a fraction [0,1] of the given target
29 
30 ##### VSD computation
31 
32 * --curve: Path to the dye curve file. If not specified, the VSD values are not
33 attenuated
34 * --depth: Depth of the attenuation curve area of influence, in micrometers. It
35 also defines the Y-coordinate at which it starts being applied, down until y=0
36 * --interpolate-attenuation: If specified, linearly interpolate the attenuation
37 values from the dye curve file. Use nearest-neighbor otherwise (default).
38 * --v0: Resting potential in millivolts
39 * --g0: Multiplier for surface area in background fluorescence term
40 * --ap-threshold: Action potential threshold in millivolts
41 * --export-volume: If specified, generate a 3D volume containing the VSD signal
42 (.mhd and .raw files)
43 * --export-point-sprites: If specified, generate a set of files describing the
44 VSD events as point sprites (.psh, .psp and .psi files)
45 
46 ##### Beer-Lambert projection
47 
48 * --sigma: Absorption + scattering coefficient (units per micrometer) in the
49 Beer-Lambert law. It must be a positive value
50 * --soma-pixels: Produce a text file with the GID, 3D position and corresponding
51 pixel coordinates in the final 2D image for all the cells loaded
52 
53 ##### Common
54 
55 * --sensor-res: Number of pixels per side of the square sensor
56 * --sensor-dim: Length of side of the square sensor in micrometers
57 * --frame[s]: Frame[s] to load in the report
58 * --time[s]: Timestamp[s] (milliseconds) to load in the report
59 * --output: Path to the directory which will store both the 2D projection data
60 and optionally the 3D volume containing the VSD values
61 
62 ## Algorithm
63 
64 The basic idea is to generate a set of events in 3D space, each of them
65 corresponding to a compartment in the neuron morphology, and then update these
66 events with the computed VSD value, based on the reported voltage from the
67 simulation.
68 
69 The 3D space will then be sampled into a homogeneous volume, resulting in a set
70 of cubic voxels, each of them containing the aggregated VSD value of all the
71 events falling within the voxel extent.
72 
73 Finally, the contents of the 3D volume will be projected vertically onto a
74 2D surface that represents the pial surface of the brain.
75 
76 All the steps in the process are detailed next.
77 
78 
79 ### Creation of the events
80 
81 The first step is to read the circuit information from the BlueConfig file,
82 using Brion, and load the morphologies corresponding to the target specified. At
83 the same time, we read the voltage report, and obtain the number of compartments
84 per section, for all the sections in each of the morphologies. With this
85 information we can compute the position of the center of the compartments,
86 by linearly interpolating the sections coordinates.
87 
88 Each of these compartments becomes an event that will eventually be updated and
89 evaluated, with an initial value of 0.
90 
91 
92 ### Calculation of the raw VSD signal from simulation data
93 
94 Then, and depending on the number of frames/times specified as a command line
95 argument, the corresponding timestamp is loaded in the voltage report. If the
96 number of compartments that is loaded from the simulation differs from the
97 number of areas in the area report, then the application will throw an
98 exception, as this is a requirement for the correct behaviour of the algorithm.
99 
100 At this point, all the events will be updated with the VSD value as follows:
101 
102 > VSD = (V - V0 + G0) * SA * AttenuationFactor
103 
104 being __V__ the corresponding voltage from the simulation report; __V0__ the
105 specified resting potential; __G0__ the specified area multiplier; and __SA__
106 the compartment's surface area.
107 
108 The value of V can be modified, if spike filtering is enabled. For experimental
109 purposes, it is possible to set an artificial threshold for the spiking
110 activity: if the input voltage goes above the specified threshold value, the
111 signal is cropped.
112 
113 > V = min(V, actionPotentialThreshold)
114 
115 The attenuation factor is a variable that depends on several factors. If a dye
116 curve file is not specified as input, it will always be 1, so no attenuation is
117 applied. It also depends on the Y-coordinate of each of the compartment and the
118 depth of the loaded circuit (modifiable by an input parameter). Its value is
119 computed as follows:
120 
121 1. The values in the input dye curve file are normalized so they go between 0
122 and 1.
123 2. If the compartment is above the specified depth value, the first attenuation
124 factor in the input dye curve file is applied.
125 3. If the compartment is below 0, the last attenuation factor in the input dye
126 curve file is applied.
127 4. If the compartment is within the specified depth, the circuit space is
128 vertically subdivided in as many regions as values are in the file. Each of the
129 attenuation factors in the file will be applied to the compartments that fall in
130 the corresponding region.
131 5. When the interpolate-attenuation option is specified, the attenuation factor
132 applied to the VSD value s the result of lineraly interpolating the two closest
133 values in the input dye curve file, so the curve is smoother.
134 
135 If the _export-point-sprites_ command line option is specified, the resulting
136 information (VSD events) will be written to disk in the form of point sprite
137 files.
138 
139 
140 ### Voxelization
141 
142 Based on two input parameters, _sensor-res_ and _sensor-dim_, we compute the
143 desired resolution for the resulting 3D volume, setting the size of the voxels.
144 In the X and Z axes, the extent of the volume corresponds to the value specified
145 in sensor-dim, in micrometers; and the number of voxels per side is determined
146 by sensor-res. The origin of the resulting volume corresponds to the center of
147 the bounding box of the soma positions.
148 
149 All the events are evaluated, computing the corresponding voxel indices based on
150 the event positions, and summing their values together when more than one share
151 a voxel index. The original floating point value is kept, being 0 the default
152 for empty voxels.
153 
154 At this point, if the _export-volume_ command line option is specified, the
155 resulting 3D volume is written to disk.
156 
157 
158 ### Projection of VSD data to the surface of brain
159 
160 To generate the projection of the data from the 3D volume onto a 2D image
161 (representing the plane that rests on the pial surface of the brain), the
162 Beer-Lambert law is used. To do that, we accumulate all the voxel values along
163 the Y direction, resulting in a 2-dimensional squared array of floating point
164 values, with _sensor-res_ pixels per side.
165 
166 For that, an ITK image filter was implemented. This filter does, for each of the
167 pixels in the final VSD projection:
168 
169 > 2DValue = sum(3Dvalue_j * exp(-sigma * depth_j))
170 
171 being __j__ the index of each of the voxels in the Y axis; __3Dvalue__ the
172 content of a specific voxel; __sigma__ a coefficient specified by the user;
173 and __depth__ the depth of the voxel in micrometers, starting from 0 at the top
174 of the volume.
175 
176 If the _soma-pixels_ command line option is specified, a text file will be
177 written to disk, containing the GID, 3D position and corresponding pixel
178 coordinates in the final 2D image for the somas of all the cells loaded.
179 
180 ## File formats
181 
182 #### Dye curve file:
183 
184 ASCII file containing all the attenuation factors, one value per line, from the
185 top to the bottom, being the first attenuation value the one that will be
186 applied to the top coordinates of the data.
187 
188 Example, attenuating more in the middle part:
189 
190 ```cpp
191 1.00
192 0.96
193 0.82
194 0.70
195 0.58
196 0.45
197 0.31
198 0.20
199 0.54
200 0.83
201 0.95
202 ```
203 
204 #### VSD point sprite files:
205 
206 A metadata header ASCII file (.psh), a binary file containing the positions for
207 all the VSD events (.psp), and one binary file containing all the event values
208 for each generated frame (.psi).
209 
210 #### Output 3D volume:
211 
212 A metadata header file (.mhd) and the binary file containing the actual volume
213 values (.raw), that can be opened from any volume rendering application, i.e.
214 Livre or Paraview.
215 
216 #### Output 2D image projection:
217 
218 A VTK file containing a metadata header section and a binary section with the
219 floating point values of the 2D projection. Readable as a NumPy array from
220 Python, as follows:
221 
222 ```py
223 import vtk
224 from vtk.util import numpy_support
225 import numpy
226 
227 reader = vtk.vtkStructuredPointsReader()
228 reader.SetFileName("image.vtk")
229 reader.ReadAllScalarsOn()
230 reader.Update()
231 
232 points = reader.GetOutput().GetPointData().GetScalars()
233 array = numpy_support.vtk_to_numpy(points)
234 ```
235 
236 #### Soma positions and pixel coordinates:
237 
238 ASCII file containing all GIDs, 3D positions and corresponding pixel coordinates
239 in the final 2D projection, for all the somas in the scene, with one cell per
240 line.
241 
242 It also includes a header section with a brief explanation of the format, file
243 version, and the library used to generate it. Example with 10 cells:
244 
245 ```cpp
246 # Soma position and corresponding pixel index for each cell, in the following format:
247 # gid [ posX posY posZ ]: i j
248 # File version: 1
249 # Fivox version: 0.6.0
250 10 [ 50.115 1971 61.384 ]: 258 256
251 20 [ 54.818 1997.1 71.462 ]: 261 262
252 30 [ 50.896 1760.8 56.659 ]: 258 254
253 40 [ 38.099 1807.9 30.819 ]: 252 241
254 50 [ 19.888 1755.2 69.948 ]: 243 261
255 60 [ 42.529 1730.5 50.998 ]: 254 251
256 70 [ 59.509 1742.3 50.691 ]: 263 251
257 80 [ 43.298 1826.1 57.443 ]: 255 254
258 160 [ 52.034 1595.8 55.536 ]: 259 253
259 170 [ 29.455 1451.9 28.081 ]: 248 239
260 ```
