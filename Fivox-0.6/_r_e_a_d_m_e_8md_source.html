4 # Introduction
5 
6 Fivox (Field Voxelization) is a library to generate volumetric images of
7 3d scalar fields (Local Field Potential, spike densities, voltage
8 sensitive dye), with loaders for the compartment, soma and spike reports
9 generated by the Neuron and NEST simulators used in the Blue Brain
10 Project. Fivox supports time animation.
11 
12 Fivox can be retrieved by cloning
13 the [source code](https://github.com/BlueBrain/Fivox).
14 
15 The voxelize command line tool can be used to generate volumes for
16 ParaView or other volume rendering applications. When compiled with
17 Livre, launch Livre with one of the URIs used by the voxelize command line tool
18 as the volume parameter. The fivox data source will be loaded
19 automatically and selected through one of the volume URI schemes.
20 
21 The sample-point command line tool can be used to extract the time series at a
22 specific 3D point. The output file can be then used as the input for the
23 plot2D.py python tool to generate a 2D graph showing the evolution of the data
24 over time.
25 
26 To use the ImageSource programmatically, please refer to the @ref fivox
27 namespace documentation and voxelize command line tool.

32 ## Features
33 
34 Fivox provides the following major features:
35 
36 * Converting compartment reports to volumetric LFP-like data
37 * Converting spike reports densities to volumetric data
38 * Converting compartment and surface area reports to volumetric data
39 * Time and animation support
40 * Extract the time series at a specific point
41 
42 # Installation
43 
44 Build Fivox from source:
45 ~~~
46 git clone https://github.com/BlueBrain/Fivox
47 mkdir Fivox/build
48 cd Fivox/build
49 cmake ..
50 make
51 ~~~

67 # About
68 
69 Fivox uses CMake to create a platform-specific build environment. The following
70 platforms and build environments are tested:
71 
72 * Linux: Ubuntu 14.04, RHEL 6.6 (Makefile, x64)
73 * Mac OS X 10.9
