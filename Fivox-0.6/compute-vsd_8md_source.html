1 compute-vsd {#compute-vsd}
2 ============
3 
4 ```
5 A tool to compute 2D projections of VSD signals using the Beer-Lambert law.:
6  -h [ --help ] Show help message
7  -v [ --version ] Show program name and version
8  --volume arg Volume URI with parameters in the form:
9  - Generic events from file: fivox://EventsFile
10  - Compartment reports: fivoxcompartments://BlueConfig?report=string&target=string
11  - Soma reports: fivoxsomas://BlueConfig?report=string&target=string
12  - Spike reports: fivoxspikes://BlueConfig?duration=float&spikes=path&target=string
13  - Synapse densities: fivoxsynapses://BlueConfig?target=string
14  - Synapse densities for pathways: fivoxsynapses://BlueConfig?preTarget=string&postTarget
15  =string
16  - Voltage-sensitive dye reports: fivoxvsd://BlueConfig?report=string&target=string
17 
18  Parameters for all types :
19  - BlueConfig: BlueConfig absolute file path (default: BBPTestData)
20  - target: name of the BlueConfig target (default: CircuitTarget)
21  - preTarget: target for presynaptic neurons for synapse densities (default: unset)
22  - postTarget: target for postsynaptic neurons for synapse densities (default: unset)
23  - gidFraction: take random cells from a fraction [0,1] of the given target (default: 1)
24  - inputMin/inputMax: minimum and maximum input values to be considered for rescaling
25  (defaults: [0.0, 2.0] for Spikes
26  [0.0, maxDensity] for Synapses
27  [-80.0, 0.0] for Compartments
28  [-15.0, 0.0] for Somas with TestData, [-80.0, 0.0] otherwise
29  [-0.0000147, 0.00225] for LFP with TestData, [-10.0, 10.0] otherwise
30  [-100000.0, 300.0] for VSD)
31  - functor: type of functor to sample the data into the voxels (defaults: 'density' for
32  Synapses, 'frequency' for Spikes, 'field' for Compartments, Somas and VSD)
33  - maxBlockSize: maximum memory usage allowed for one block in bytes (default: 64MB)
34  - cutoff: the cutoff distance in micrometers (default: 100)
35  - extend: the additional distance, in micrometers, by which the original data volume
36  will be extended in every dimension (default: 0, the volume extent matches the bounding
37  box of the data events). Changing this parameter will result in more volumetric data, and
38  therefore more computation time
39  - reference: path to a reference volume to take its size and resolution, overwrites the
40  'size' and 'resolution' parameter
41  - size: size in voxels along the largest dimension of the volume, overwrites the
42  'resolution' parameter
43  - resolution: number of voxels per micrometer (default: 0.0625 for densities, otherwise
44  0.1)
45 
46  Parameters for Compartments:
47  - report: name of the compartment report (default: 'voltage'; 'allvoltage' if BlueConfig
48  is BBPTestData)
49  - dt: timestep between requested frames in milliseconds (default: report dt)
50 
51  Parameters for Somas:
52  - report: name of the soma report (default: 'soma'; 'voltage' if BlueConfig is
53  BBPTestData)
54  - dt: timestep between requested frames in milliseconds (default: report dt)
55 
56  Parameters for Spikes:
57  - duration: time window in milliseconds to load spikes (default: 10)
58  - spikes: path to an alternate out.dat/out.spikes file (default: SpikesPath specified in
59  the BlueConfig)
60 
61  Parameters for VSD:
62  - report: name of the voltage report (default: 'soma'; 'voltage' if BlueConfig is
63  BBPTestData)
64  - areas: path to an area report file (default: path to TestData areas if BlueConfig is
65  BBPTestData)
66  - dt: timestep between requested frames in milliseconds (default: report dt)
67  -t [ --time ] arg Timestamp to load in the report
68  --times arg Time range [start end) to load in the report
69  -f [ --frame ] arg Frame to load in the report
70  --frames arg Frame range [start end) to load in the report
71  -o [ --output ] arg Name of the output volume (if 'export-volume' specified) and 2D image (VTK format) files;
72  contains frame number if --frames or --times specified.
73  --export-volume Export also the 3d volume (mhd + raw) containing the VSD values, in addition to the VTK
74  file.
75  --export-point-sprites Export also the point sprite files containing the VSD values, in addition to the VTK file.
76  --sensor-res arg (=512) Number of pixels per side of square sensor.
77  --sensor-dim arg (=1000) Length of side of square sensor in micrometers.
78  --curve arg Path to the dye curve file (default: no attenuation)
79  --depth arg (=2081.75635) Depth of the attenuation curve area of influence. It also defines the Y-coordinate at
80  which it starts being applied, down until y=0 (default: 2081.75641787 micrometers).
81  --interpolate-attenuation Interpolate the attenuation from the dye curve.
82  --sigma arg (=0.0045) Absorption + scattering coefficient (units per micrometer) in the Beer-Lambert law. Must
83  be a positive value (default: 0.0045).
84  --v0 arg (=-65) Resting potential (default: -65 mV).
85  --g0 arg (=0) Multiplier for surface area in background fluorescence term.
86  --ap-threshold arg Action potential threshold in millivolts.
87  --soma-pixels arg Produce a text file containing the GIDs loaded and their corresponding 3D positions and
88  indices in the resulting 2D image.
