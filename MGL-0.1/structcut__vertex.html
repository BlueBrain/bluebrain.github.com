Detailed Description

a version of vertex which is a cut_tangle 

\remark for the migration function to work efficiently,
we had to add a migrating_vertex pointer. This is not
a performance killer, but it is suspiciously out of place.
Any suggestions on how to remove it are welcome.

the cut_vertex parallelizes vertex by adding the ability to store vertex_child structures that represent remote children and cut_edge structures that represent edges cut across parts. These new structures all contain attribute data, which is synchronized in the parallel collective operations.
