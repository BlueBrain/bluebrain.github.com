10 # Release 0.8.0 (09-Dec-2016)
11 
12 ### Enhancements
13 
14 * An optimization called alpha cut-off has been implemented in
15  FragmentListOITBin. This optimization consist on discarding fragments known
16  to be occluded due to alpha accumulation. Alpha accumulation is computed
17  in an order independent manner, so the effectiveness is very scene dependent.
18  In particular this is very useful in scenes with high depth complexity and
19  relatively high average alpha (> 0.5).
20  The optimization can be enabled with a new parameter to the algorithm.
21 * GPU timing added to FragmentListOITBin for profiling.
22 
23 ### API Changes
24 
25 * Changed FragmentListOITBin::Parameters to replace the callback variable
26  with a setter function that allows a different callback per context.
27 * The signature of the fragment capture callback function has been changed to
28  take a class with accessor methods instead of a collection of textures.
29 * FragmentListOITBin::Parameters::sortAndDisplay removed, the capture callback
30  has been modified to use its return value instead of a member variable.
31 * New functions added to FragmentListOITBin::Parameters to enable and disable
32  alpha cut-off.

68 # Release 0.5.0 (3-Oct-2104)
69 
70 ### New Features
71 
72 * The opacity threshold parameter in multi-layer depth peeling is now used
73  to choose a simplified code path that doesn't check accumulated alpha
74  when its value is >= 1.0.
75 
76 ### API Changes
77 
78 * MultiLayerDepthPeelingBin::Parameter::opacityThreshold changed to be a
79  constant parameters that must be given at construction time. Setting a new
80  prototype bin with a different threshold value will require shader
81  compilation at the next render traversal because shaders cannot be reused.

145 # Release 0.2.0 (21-Mar-2013)
146 
147 ### Enhancements
148 
149 * During rendering, a shading programs is associated to each state set found
150  in the scenegraph. Previously, it was mandatory to provide shaders for each
151  state set using BaseRenderbin::addExtraShadersForState. Now default shaders
152  are created for those state sets the user hasn't provide any shaders. It is
153  an error to provide shaders which don't follow the specification described
154  in addExtraShadersForState.
155

176 # Release 0.1.0 (4-Dec-2012)
177 
178 This is the first release of OSG Transparency, a library that provides 3 render
179 bins for transparent geometry.
180 
181 ### Known bugs
182 
183 * The rendering of opaque and transparent geometry cannot be mixed. Objects
184  rendered with OSG Transparency will not be in correct depth with objects
185  rendered by other render bin.
