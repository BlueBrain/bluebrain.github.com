4 ## Deflect 0.13
5 
6 ### Deflect 0.13.0 (10-05-2017)
7 * [163](https://github.com/BlueBrain/Deflect/pull/163):
8  Stereo streaming accepts side-by-side images as input.
9 * [161](https://github.com/BlueBrain/Deflect/pull/161):
10  DesktopStreamer OSX: the default hosts are read from a json file in the app
11  bundle. This is useful for external users who want to adapt the official
12  BlueBrain releases to their own needs without recompiling.
13 * [156](https://github.com/BlueBrain/Deflect/pull/156):
14  Deflect can be released as a Debian/Ubuntu package. Once installed,
15  DesktopStreamer can be launched from the desktop's applications menu.
16 * [153](https://github.com/BlueBrain/Deflect/pull/153):
17  - Fix: Stream async send used LIFO queueing, not FIFO
18  - Optimize concurrency of Stream asynchronous send
19 * [157](https://github.com/BlueBrain/Deflect/pull/157):
20  Better event registration mechanism, fixes warnings in the multi-client case.
21 * [154](https://github.com/BlueBrain/Deflect/pull/154):
22  On the server side, segments can be decoded to YUV images, saving CPU time by
23  skipping the YUV -> RGB conversion step.
24  Additionally, Stream images can be further compressed using optional
25  [chroma subsampling](ChromaSubsampling.md).
26 * [148](https://github.com/BlueBrain/Deflect/pull/148),
27  [153](https://github.com/BlueBrain/Deflect/pull/153):
28  Support for streaming stereo 3D content in a frame-sequential manner.

42 ### 0.12.0 (09-12-2016)
43 * [143](https://github.com/BlueBrain/Deflect/pull/143):
44  The non-functional Servus dependency has been removed.
45 * [142](https://github.com/BlueBrain/Deflect/pull/142):
46  The DesktopStreamer app uses the full user name as the default stream name.
47 * [139](https://github.com/BlueBrain/Deflect/pull/139):
48  OSX: AppNap is now disabled for all QmlStreamers. The AppNapSuspender class
49  is also available in Deflect library for use in external applications.
50 * [137](https://github.com/BlueBrain/Deflect/pull/137):
51  Deflect Qt: the offscreen Qml view used by the Qml streamer is available as a
52  separate class.
53 * [133](https://github.com/BlueBrain/Deflect/pull/133):
54  QmlStreamer: Use asynchronous rendering, add
55  deflect::qt::QmlStreamer::useAsyncSend() to enable asynchronous image streaming
56  (off by default).
57 * [130](https://github.com/BlueBrain/Deflect/pull/130):
58  Replaced boost by C++11. Boost is now an optional dependency and it is used
59  only by the tests. Some API changes were introduced by this change.
60 * [129](https://github.com/BlueBrain/Deflect/pull/129):
61  Cleared Deflect from boost::serialization that was used exclusively by Tide.
62 * [128](https://github.com/BlueBrain/Deflect/pull/128):
63  New events for transmitting all touch points in addition to existing gestures:
64  - Gives the ability to handle more than one touch point in applications (e.g.
65  draw with multiple fingers on a whiteboard).
66  - Simplifies integration for applications that natively handle multitouch
67  events, like Qml-based ones.
68  - DeflectQt uses a new TouchInjector class for sending touch events to Qt
69  (moved from Tide).
70  In addition:
71  - A new *pinch* event was added. Clients of Tide >= 1.2 have to adapt their
72  code to use it instead of *wheel* events, which are no longer sent.
73  - Minor additions to the simplestreamer demo application.
74 * [126](https://github.com/BlueBrain/Deflect/pull/126):
75  DesktopStreamer: The list of default hosts can be configured using the CMake
76  variable DEFLECT_DESKTOPSTREAMER_HOSTS.
77 * [124](https://github.com/BlueBrain/Deflect/pull/124):
78  QmlStreamer: Users can now interact with WebGL content in a WebEngineView
79  and no longer risk opening a system context menu with a long press (prevent
80  crashes in certain offscreen applications).
81 * [123](https://github.com/BlueBrain/Deflect/pull/123):
82  QmlStreamer is now compatible with Qml WebEngineView items. Users must call
83  QtWebEngine::initialize() in their QApplication before creating the stream.
84  To receive keyboard events, the objectName property of a WebEngineView must
85  be set to "webengineview".
86  QmlStreamer also exposes the Stream::sendData() function and swipe gestures
87  are available in Qml from a "deflectgestures" context property.
88 * [119](https://github.com/BlueBrain/Deflect/pull/119):
89  Added deflect::Stream::sendData() to allow sending user-defined information
90  to the deflect::Server.
91 * [118](https://github.com/BlueBrain/Deflect/pull/118):
92  New "pan" event for multi-finger pan gestures.
93 * [117](https://github.com/BlueBrain/Deflect/pull/117):
94  Fix several issues with stream connection / disconnection [DISCL-375]
95 * [115](https://github.com/BlueBrain/Deflect/pull/115):
96  Bugfix for possible endless loop of warnings in ServerWorker::_flushSocket():
97  "QAbstractSocket::waitForBytesWritten() is not allowed in UnconnectedState"
98 * [114](https://github.com/BlueBrain/Deflect/pull/114):
99  QmlStreamer: added support for keyboard events

126  DesktopStreamer: windows that are streamed independently are activated

129 * [100](https://github.com/BlueBrain/Deflect/pull/100):
130  QmlStreamer: correcly quit application when stream is closed.

132  Fix incomplete socket send under certain timing conditions
133 * [98](https://github.com/BlueBrain/Deflect/pull/98):
134  Streams can be constructed based on the DEFLECT_ID and DEFLECT_HOST ENV_VARs.
135 * [97](https://github.com/BlueBrain/Deflect/pull/97):
136  DesktopStreamer: the list of windows available for streaming is also updated
137  after a window has been hidden or unhidden.
138 * [94](https://github.com/BlueBrain/Deflect/pull/94):
139  Removed legacy SendCommand functionality from Stream to Server. It was only
140  used internally by the old Tide Dock streamer and never meant to be public.
141 * [93](https://github.com/BlueBrain/Deflect/pull/93):
142  Minor fixes for the Qml stream API:
143  - Cleanup size handling, allow programmatic resizing of root Qml item
144  - Do not send a size hints event if none were set by the user
