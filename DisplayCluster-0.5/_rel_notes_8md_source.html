4 # New in this release {#New}
5 
6 DisplayCluster 0.5 provides the following improvements:
7 
8 ## New Features {#NewFeatures}
9 
10 * The dcStream library has been moved to a separate project:
11  [Deflect](https://github.com/BlueBrain/Deflect).
12 * The project has been ported from Qt4 to Qt5
13 * Compilation of external dependencies (Deflect, TUIO) has been simplified
14  thanks to the "git subproject" functionality. Dependencies are cloned during
15  the CMake configure step and built alongside the project.
16 * Rendering of contents is done in QML for a better and simplified UI design.
17 * Windows have a title bar which shows the name of the file (or the stream).
18 * Windows have customizable action buttons which allow users to close / maximize
19  windows and pause movies from the wall.
20 * Windows have dragable borders for reszing them from the wall.
21 * Synchronized buffer swap and vsync works for all types of window settings.
22 * Added pyramidmaker utility to create image pyramids from the command line.
23 * Image pyramids keep the format of the source image file instead of using jpg.
24 * Alpha blending (transparency) for SVGs and appropriate image types.
25 
26 ## Enhancements {#Enhancements}
27 
28 * The startdisplaycluster script has been greatly improved and handles more MPI
29  implementations (OpenMpi, MVAPICH, MPICH2, MPICH v3).
30 * Window move and resize operations go through a controller to constrain them
31  inside a valid range.
32 * Resizing and zooming inside windows happens around the mouse or touch gesture
33  center.
34 * FPS statistics of PixelStreams are per-stream instead of per-segment.
35 * Debug display of PixelStream segements show the absolute size of the segments.
36 * A few extra unit tests (StateSerialization, ContentWindowController,
37  DoubleTapGesture).
38 * Multiple refactorings and code cleanups. All global variables have been
39  removed.
40 * CMake cleanups, using common_library and common_application from CMake/common.
41 * Improved clarity of log messages
42 
43 ## Optimizations {#Optimizations}
44 
45 * PDFs and SVGs have been optimized to move and resize much faster.
46 * PDFs and SVGs don't redraw constantly in zoomed mode or in settings with
47  mutliple windows per node.
48 
49 ## Documentation {#Documentation}
50 
51 * Added a short user guide.
52 * Removed obsolete manual.pdf.
53 
54 ## Bug Fixes {#Fixes}
55 
56 * The Webbrowser widget uses the correct webpage from the configuration.
57 * Improved support for decoding Movies.
58 * Fix session loading from legacy xml.

62 # New in DisplayCluster 0.4
63 
64 DisplayCluster 0.4 provides the following improvements:
65 
66 ## New Features
67 
68 * Movies play synchronously across all screens.
69 * The launch procedure relies on a single python script that works out of the
70  box on all supported platforms and with the different MPI implementations. It
71  is compatible with both Python 2.x and 3.x APIs.
72 * dc::Stream implements an asyncSend() function for PixelStreams.
73 * MPIBenchmark application to measure interprocess communication performance.
74 
75 ## Enhancements
76 
77 * DesktopStreamer properly handles AppNap on OSX 10.9.
78 * DesktopStreamer detects Retina displays automatically (no retina checkbox).
79 * ImagePyramids use a correct aspect ratio instead of a square size, leading to
80  a significant reduction in disk space usage for newly generated pyramids.
81 * MasterWindow UI menus have been reorganized. Help -> about shows the git
82  revision used for the build.
83 * The DisplayGroup, ContentWindow and related classes have undergone
84  profound refactorings and now follow an MVC pattern.
85 * Use of global variables has been drastically reduced.
86 * The CMakeLists.txt follows common Eyescale/CMake conventions.
87 * MPI message headers are now separate from dc::Stream message headers.
88 * The OpenGL rendering code has been improved, but still needs more attention.
89 
90 ## Optimizations
91 
92 * Multithreaded MPI communication between the processes brings significant
93  performance improvements.
94 * PixelStream dispatch rate from master application is based on feedback from
95  wall applications for a correct flow control.
96 * The handling of Content dimensions is greatly simplified and the
97  ContentDimensionsRequest has been removed.
98 
99 ## Documentation
100 
101 * Separate Introduction page from Release Notes for more clarity.
102 
103 ## Bug Fixes
104 
105 * Fixed DynamicTexture LOD mechanism. Textures are now displayed at the correct
106  resolution.
107 * Fixed startup crash with an empty background.
108 * Fixed crashes when opening incorrect State files.
109 * Fix crash on streaming client exit, fix signaling of new events.
110 * Fixed multi-source PixelStream intialization.
111 * All the exit(-1) calls have been replaced by std::exceptions.
112 * Fixed all cppcheck warnings.
113 * Fixed build with older FFMPEG versions.
114 
115 ## Known Bugs
116 
117 The following bugs were known at release time:
118 * Sometimes, once closed the dock may refuse to open again ("Already have a
119  window for stream: Dock").
120 * Touch events may occasionnaly pass through the active window onto the
121  background.
