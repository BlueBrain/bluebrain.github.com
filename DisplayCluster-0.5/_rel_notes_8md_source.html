4 # New in this release {#New}
5 
6 DisplayCluster 0.5 provides the following improvements:
7 
8 ## New Features {#NewFeatures}
9 
10 * The dcStream library has been moved to a separate project:
11  [Deflect](https://github.com/BlueBrain/Deflect).
12 * Compilation of external dependencies (Deflect, TUIO) has been simplified
13  thanks to the "git subproject" functionality. Dependencies are cloned during
14  the CMake configure step and built alongside the project.
15 * Rendering of contents is done in QML for a better and simplified UI design.
16 * Windows have customizable action buttons which allow users to close / maximize
17  windows and pause movies from the wall.
18 * Windows have dragable borders for reszing them from the wall.
19 * Synchronized buffer swap and vsync works for all types of window settings.
20 
21 ## Enhancements {#Enhancements}
22 
23 * The startdisplaycluster script has been greatly improved and handles more MPI
24  implementations (OpenMpi, MVAPICH, MPICH2, MPICH v3).
25 * Window move and resize operations go through a controller to constrain them
26  inside a valid range.
27 * Resizing and zooming inside windows happens around the mouse or touch gesture
28  center.
29 * FPS statistics of PixelStreams are per-stream instead of per-segment.
30 * Debug display of PixelStream segements show the absolute size of the segments.
31 * A few extra unit tests (StateSerialization, ContentWindowController,
32  DoubleTapGesture).
33 * Multiple refactorings and code cleanups. All global variables have been
34  removed.
35 
36 ## Optimizations {#Optimizations}
37 
38 * PDFs and SVGs have been optimized to move and resize much faster.
39 * PDFs and SVGs don't redraw constantly in zoomed mode or in settings with
40  mutliple windows per node.
41 
42 ## Documentation {#Documentation}
43 
44 * Added a short user guide.
45 * Removed obsolete manual.pdf.
46 
47 ## Bug Fixes {#Fixes}
48 
49 * The Webbrowser widget uses the correct webpage from the configuration.
50 * Improved support for decoding Movies.
51 * Fix session loading from legacy xml.

55 # New in DisplayCluster 0.4
56 
57 DisplayCluster 0.4 provides the following improvements:
58 
59 ## New Features
60 
61 * Movies play synchronously across all screens.
62 * The launch procedure relies on a single python script that works out of the
63  box on all supported platforms and with the different MPI implementations. It
64  is compatible with both Python 2.x and 3.x APIs.
65 * dc::Stream implements an asyncSend() function for PixelStreams.
66 * MPIBenchmark application to measure interprocess communication performance.
67 
68 ## Enhancements
69 
70 * DesktopStreamer properly handles AppNap on OSX 10.9.
71 * DesktopStreamer detects Retina displays automatically (no retina checkbox).
72 * ImagePyramids use a correct aspect ratio instead of a square size, leading to
73  a significant reduction in disk space usage for newly generated pyramids.
74 * MasterWindow UI menus have been reorganized. Help -> about shows the git
75  revision used for the build.
76 * The DisplayGroup, ContentWindow and related classes have undergone
77  profound refactorings and now follow an MVC pattern.
78 * Use of global variables has been drastically reduced.
79 * The CMakeLists.txt follows common Eyescale/CMake conventions.
80 * MPI message headers are now separate from dc::Stream message headers.
81 * The OpenGL rendering code has been improved, but still needs more attention.
82 
83 ## Optimizations
84 
85 * Multithreaded MPI communication between the processes brings significant
86  performance improvements.
87 * PixelStream dispatch rate from master application is based on feedback from
88  wall applications for a correct flow control.
89 * The handling of Content dimensions is greatly simplified and the
90  ContentDimensionsRequest has been removed.
91 
92 ## Documentation
93 
94 * Separate Introduction page from Release Notes for more clarity.
95 
96 ## Bug Fixes
97 
98 * Fixed DynamicTexture LOD mechanism. Textures are now displayed at the correct
99  resolution.
100 * Fixed startup crash with an empty background.
101 * Fixed crashes when opening incorrect State files.
102 * Fix crash on streaming client exit, fix signaling of new events.
103 * Fixed multi-source PixelStream intialization.
104 * All the exit(-1) calls have been replaced by std::exceptions.
105 * Fixed all cppcheck warnings.
106 * Fixed build with older FFMPEG versions.
107 
108 ## Known Bugs
109 
110 The following bugs were known at release time:
111 * Sometimes, once closed the dock may refuse to open again ("Already have a
112  window for stream: Dock").
113 * Touch events may occasionnaly pass through the active window onto the
114  background.
