4 # Release 1.7.0 (24-Mar-2016)
5 
6 * [56](https://github.com/BlueBrain/Brion/pull/56):
7  Improved target parser

11  Added the method brain::Circuit::getRotations
12 * [46](https://github.com/BlueBrain/Brion/pull/46):
13  Fixed a bug in enum to string conversions affecting morphologyConverter.
14 * [45](https://github.com/BlueBrain/Brion/pull/45):
15  Made targets optional.
16  - brain::Circuit::getGIDs() returns all GIDs handled by the circuit.
17  - brain::Circuit::getGIDs( target ) returns GIDS for a specified target.
18  - brain::Circuit::getNumNeurons() returns the total number of neurons in the
19  circuit.

21  Add MVD3 support to brain::circuit

23  Add compartment report converter tool
24 * [30](https://github.com/BlueBrain/Brion/pull/30),
25  [35](https://github.com/BlueBrain/Brion/pull/35):
26  Added a new library, Brain, to provide higher level functions and classes.
27  The library provides:
28  - A Circuit class to get basic information from cells and targets and load
29  morphologies at circuit level.
30  - A brain::cell namspace with a Morphology class plus other related classes
31  to access morphological data about neurons.

58  points, these types are translated into the most reasonable one based on the

61 # Release 1.5.0 (7-Jul-2015)
62 
63 * Add RESTING_VOLTAGE constant.
64 * Add support for binary spike reports.
65 * Add support for spike writer plugin.
66 * Add async IO support in brion::CompartmentReportMap.
67 * brion::CompartmentReportBinary::updateMapping() more robust.
68 * brion::SpikeReport API extended to support stream-based reports. A reference
69  implementation of a stream-based spike report plugin is also provided.
70 * Ensure thread-safety with non-threadsafe HDF5 library.
71 * Morphology class implementation changed to support the Lunchbox plugin
72  mechanism.
73 * New morphology plugin to read SWC morphologies.
74 * [#3](https://github.com/BlueBrain/Brion/pull/3):
75  New morphology converter tool.
76 * New null compartment report for benchmarks.
77 * New spike converter tool.
78 * Optimized function Circuit::get for large circuits (cut down the loading time
79  of the 3.1 M neuron circuit by 6.5 seconds in a Core i7-4930K @ 3.40GHz).
80 * Replace use of iostreams by lunchbox::MemoryMap.
81 * Several bugfixes and cleanups.
82 
83 # Release 1.4.0 (8-Oct-2014)
84 
85 * Old report readers have been adapted to be static plugins using
86  lunchbox::IOPluginFactory.
87 * New spike report reader plugin for NEST spike reports. The plugin can
88  parse multiple report file using shell wildcards (*, ?) in the
89  filepath provided.
90 * New constructor for brion::CompartmentReport accepting an URI and the desired
91  access mode deprecates the old read and write constructors.
92 * New compartment report based on key-value stores supported by
93  Lunchbox::PersistentMap.
94 * Spike loading has been optimized (in a 12-core GPU the expected
95  speed-up is above 4x for large files).
