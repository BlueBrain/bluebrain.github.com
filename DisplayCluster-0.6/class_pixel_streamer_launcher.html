Detailed Description

Launch Pixel Streamers as separate processes.

The processes connect to the Master application on localhost using the deflect::Stream API. They can be terminated by the user by closing their associated window.

Due to an incompatibility between QProcess and MPI(*), we must start the processes DETACHED. In theory they might stay alive after the main application has exited. In practice, this doesn't seem to happen; our processes exit when the deflect::Stream is closed in any case.
