Detailed Description

Stream visual data to a deflect::Server.

A Stream can be subdivided into one or more images and eye passes. This allows to have different applications each responsible for sending one part of the global image.

The methods in this class are reentrant (all instances are independant) but are not thread-safe.

Open a new connection to the Server.

The user can check if the connection was successfully established with isConnected().

Different Streams can contribute to a single window by using the same identifier. All the Streams which contribute to the same window should be created before any of them starts sending images.

Parameters
idThe identifier for the stream. If left empty, the environment variable DEFLECT_ID will be used. If both values are empty, a random unique identifier will be used.
hostThe address of the target Server instance. It can be a hostname like "localhost" or an IP in string format like "192.168.1.83". If left empty, the environment variable DEFLECT_HOST will be used instead.
portPort of the Server instance, default 1701.
Exceptions
std::runtime_errorif no host was provided or no connection to server could be established

Future deflect::Stream::finishFrame ( )

Asynchronously notify that all the images for this frame have been sent.

This method must be called everytime this Stream instance has finished sending its image(s) for the current frame. The receiver will display the images once all the senders which use the same identifier have finished a frame. This is only to be called once per frame, even for stereo rendering.
