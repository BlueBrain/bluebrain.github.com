Detailed Description

a cut version of tree based on cut_cushion

the structures and functions in cut_pin_cushion.h implement a one-to-many relationship between objects distributed across parts, which enables the tree structure to be distributed across parts as well. Although we try to make this as elegant as possible, several annoying asymmetries show up when distributing tree.

Chief amongst these is the concept of orphans. Orphans are branches whose parent branch exists on a different part. They cannot directly link to it because it is on a different part. Orphans come about when the parent/child link (represented by the link between a pin and a cushion) is broken (resulting in a cushion/needle link and a cross-part needle/pin_head link). The first problem with orphans is switching the so-called "parent" part of the branch structure (the one that links it to its parent) from a pin to a pin_head. To implement this we use a union and a boolean. It may be cleaner to have a different structure for an orphan, but that would break things in places higher up like the multiscale migration algorithm, where addresses are expected to remain constant.
