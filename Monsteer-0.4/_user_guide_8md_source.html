6 # Overview {#Overview}
7 
8 The Monsteer project is a set of libraries and tools that helps users to visualize,
9 analyse and steer their simulations. Currently, the target community are NEST
10 users who write simulators using the PyNEST interface. These users generally
11 want to visualize the neural activity and and possibly steer the simulation
12 accordingly. The Monsteer library fills this gap, providing the necessary mechanisms
13 to stream and steer the simulation results through user-friendly python modules.
14 
15 # Compilation {#Compilation}
16 
17 Minimum configuration to configure using cmake, compile and run Monsteer:
18 
19 * A Linux box,
20 * GCC compiler 4.8+,
21 * CMake 2.8+,
22 * Boost 1.54,
23 * MPI (OpenMPI, mvapich2, etc),
24 * NEST simulator 2.4.2,
25 * MUSIC 1.0.7,
26 * Python 2.6,
27 
28 # Setup {#Setup}
29 
30 After the Monsteer project is compiled and installed, an executable called
31 *music_proxy* and the Python packages *bbp* and *Monsteer* will be present in
32 the installation directory.
33 
34 Before running music configurations, the environmental variables PATH and
35 PYTHONPATH must be configured to point to the location of the music_proxy binary
36 and the Python packages *bbp* and *Monsteer* respectively. To visualize spike
37 activity, the RTNeuron application has to be installed as well.
38 
39 Sample scripts and configuration files for steering and streaming are provided
40 in the *share/monsteer/examples/nest* directory from the installation tree.
41

56 ## Music Configuration {#Music_Configuration}
57 
58 In the share/monsteer/examples/nest directory there are files with the extension
59 ".music" and there are NEST simulation scripts. MUSIC configurations describe
60 how to launch multiple applications that will communicate using MUSIC, how many
61 MPI processes a MUSIC application will use and how its ports should be connected.
62 
63 Below is a sample music configuration:
64 
65 ~~~
66 # Usage:
67 # - With mpirun, form this directory do:
68 # $ mpirun -np 5 music nest2music_proxy_with_steering.music
69 # - With srun
70 # srun -n 5 music nest2music_proxy_with_steering.music
71 
72 stoptime=10000000
73 
74 [mars]
75  binary=./minimal_steered.py
76  np=4
77 
78 [venus]
79  # This binary is expected to be in the path
80  binary=music_proxy
81  args=--steering
82  np=1
83 
84 mars.spikes_out -> venus.spikesPort [1000]
85 venus.steeringPort -> mars.steering_input [0]
86 ~~~
87 
88 In this configuration:
89 
90 Sections:
91 * "stoptime" is the total simulation time given in seconds,
92 * "binary" is an executable name,
93 * "np" is the number of MPI processes,
94 * "args" contains the command line arguments.
95 
96 Options:
97 * "mars" and "venus" are application definitions which will be executed using
98 MUSIC environment.
99 
100 Connection definitions:
101 * "mars.spikes_out -> venus.spikesPort [1000]" constructs the "spike event"
102  connection from the "mars" application to the "venus" application. "1000" is
103  the number of channels this connection has. In the Monsteer case, there are
104  1000 neurons that reports their spike activity.
105 * "venus.steeringPort -> mars.steering_input [0]" connects "messaging"
106  ports. "0" means that there will be only one channel to communicate. This
107  parameter is used for setting up the messaging ports.
108 
109 Please refer to the [MUSIC](http://incf.org/activities/our-programs/modeling/music)
110 documentation for more detailed documentation.

112 ## NEST script {#Nest_script}
113 
114 The NEST script is responsible for running the neural simulation. For the NEST
115 simulator to communicate to the "music_proxy" application, the spike events and
116 steering MUSIC ports must be connected to it. The script below setups the MUSIC
117 communication ports and runs the simulation.
118 
119 \code{.py}
120 duration = 100000
121 cell_count = 1000
122 
123 import nest
124 
125 neurons = Create("iaf_neuron", cell_count)
126 
127 # Setups music spike output.
128 music_spike_output = Create('music_event_out_proxy', 1)
129 SetStatus(music_spike_output, {'port_name': 'spikes_out'})
130 
131 # Connecting neurons to music event channels.
132 for i in range(0, len(neurons)):
133  Connect([neurons[i]], music_spike_output, 'one_to_one',
134  {'music_channel': i+1})
135 
136 # Setup music steering input.
137 music_steering_input = _nest.Create('music_message_in_proxy', 1)
138 SetStatus(music_steering_input,
139  {'port_name': 'steering_input',
140  "acceptable_latency": 40.0})
141 
142 time = 0
143 while time < duration:
144  Simulate(10)
145  time += 10
146 \endcode
147 
148 Note that the code above does not include the handling of steering messages,
149 but Monsteer provides a @ref Nesteer python module for that purpose.
150 
151 ## Nesteer {#Nesteer}
152 
153 The Nesteer python class helps users to setup MUSIC communication and map BBP
154 circuits to NEST simulations. The example below simplifies the previous NEST
155 script by making use of the Nesteer steering message parser.
156 
157 \code{.py}
158 duration = 100000 #In miliseconds.
159 cell_count = 1000
160 
161 from monsteer.Nesteer import Nesteer
162 from nest import *
163 
164 neurons = Create("iaf_neuron", cell_count)
165 
166 monsteer_communicator = Nesteer('spikes_out',
167  'steering_input',
168  neurons,
169  [i + 1 for i in range(cell_count)])
170 
171 def apply_spikes(generator, neurons):
172  Connect(generator, neurons, 'all_to_all', syn_spec={'weight': 40.0})
173 monsteer_communicator.set_apply_generators_to_neurons_function(apply_spikes)
174 
175 time = 0
176 while time < duration:
177  Simulate(10)
178  monsteer_communicator.process_events()
179  time += 10
180 \endcode
181 
182 The parameters of the Nesteer class:
183 
184 - 'spikes_out' is MUSIC spike event port name,
185 - 'steering_input' is MUSIC messaging port name for steering,
186 - 'neurons' is the Nesteer neurons.
187 - '[i + 1 for i in range(cell_count)]' defines which neuron corresponds to
188  which channel on the event port. In this example, the event ports are
189  connected to each one of the 1000 neurons, and each of them is mapped to every
190  event channel from 1 to 1000. This corresponds to the number of MUSIC event
191  channels.
192 
193 In the case of BBP circuits, it is the mapping of NEST neuron ids to BBP neuron
194 ids, because NEST and BBP have different ids for cells. To take BBP circuits
195 into account, the event channels in MUSIC configuration should be set to the
196 maximum neuron id.

198 # Working with the Simulation {#Working_With_Simulation}
199 
200 After setting up the configuration and script files, the simulation can be
201 executed using either mpirun or srun depending on the MPI environment setup.
202 Assuming the MUSIC configuration file is named "config.music", and the total
203 number of processes configured in this file is "5", then it can either be run
204 with mpirun or srun as follows:
205 
206 ~~~
207 srun -n 5 music config.music
208 ~~~
209 or
210 ~~~
211 mpirun -np 5 music config.music.
212 ~~~
213 
214 This will trigger running of the simulation and sharing information with the
215 "music_proxy" application.

217 ## Steering {#Steering}
218 
219 To send steering commands to the simulation, there is a python NEST helper
220 object. Through this object, it is possible to inject different spike
221 generators to neurons, playing and pausing the simulation. The next example
222 demonstrates the injection of stimuli to multiple neurons and then pausing
223 the simulation.
224 
225 \code{.py}
226 
227 import monsteer.Nesteer
228 import bbp
229 import nest
230 import numpy
231 import time
232 
233 # Creating the steering connection
234 simulator = monsteer.Nesteer.Simulator('monsteer_nesteer://')
235 
236 # Get parameter dictionary for a spike generator
237 params = nest.GetDefaults('poisson_generator')
238 
239 # Modify rate
240 params['rate'] = 50000.0
241 
242 # Define which cells will be effected by the spike generator
243 cells = bbp.Cell_Target()
244 for i in range(1000):
245  cells.insert(i + 1)
246 
247 time.sleep(4) # Waiting for connection
248 
249 # Inject stimulus to the simulator
250 simulator.injectStimulus(params, cells)
251 time.sleep(4) # Waiting for connection
252 
253 # Pause the simulator
254 simulator.pause()
255 \endcode

257 ## Spike Visualization {#Spike_visualization}
258 
259 To visualize the spikes of a BBP circuit, the RTNeuron circuit visualisation
260 tool can be employed. It has the capability to connect to a running simulation and
261 visualize spike events in the circuit.
262 
263 In the example below, a BlueBrain circuit is utilized. The configuration file
264 for the circuit can be accessed from the lugano cluster at:
265 "/gpfs/bbp.cscs.ch/apps/viz/bbp/dev/BBPTestData/master/BBPTestData/Build/include/BlueConfig".
266 
267 In this circuit there are 1000 neurons. The sample simulation scripts in the
268 source code repository in Monsteer can be employed to run simulations on them.
269 
270 After starting the simulation with a MUSIC configuration, a user can launch
271 RTNeuron with the command line below. The --gui parameter activates the gui,
272 which can then be used to connect to a running simulation by clicking the
273 "OpenSimulation" button.
274 
275 ~~~
276 rtneuron-app.py -b "BLUE CONFIG FILE" --target Column soma --neurons 1 1000 soma --shell --gui [ -s monsteer:// ]
277 ~~~
278 
279 If the GUI is enabled, clicking on the "OpenSimulation" button brings a dialog
280 for entering the url for the music proxy spike port. If there is only one
281 simulation running, simply "monsteer://" can be entered, else "monsteer://host:port"
282 should be entered for the right host and port which provides the spikes. If
283 GUI is not enabled, "-s monsteer://" parameter can be added to the RTNeuron
284 execution command line.
285 
286 Once the connection is established, spike events should be visible on the
287 screen. Interacting with the GUI item "spike tail" slider changes how
288 long a spike is visible while the "simulation delta" modifies the the speed of
289 the simulation.
290 
291 If the connection was established through a command line parameter, some
292 additional commands must be executed in the python shell to start
293 visualizing the spikes or to adjust the spike tails and simulation delta.
294 Those commands can be seen below:
295 
296 \code{.py}
297 app.player.play()
298 app.player.simulationDelta = 0.1
299 view.attributes.spike_tail = 0.01
300 \endcode
