27 
28 @snippet CMakeLists.txt Setup
29 
30 The GitExternal script clones and updates the given git repository to
31 the given revision. To update the revision to the latest master, it also
32 provide a 'make update' target.
33 
34 - - -
35 
36 Before including the Common cmake setup, the project versions are
37 defined:
38 
39 @snippet CMakeLists.txt Versioning
40 
41 The API version is composed of major.minor.patch. Major is increased
42 every time the API is broken, that is, users of the software need to
43 change something in their implementation to compile. Minor is increased
44 for each release with new features or optimizations which do not break
45 the official programming interface. Unstable prototype software uses 0
46 as major and is allowed to break the API between minor versions (since
47 it does not have an official 1.0 API). A patch release fixes only bugs
48 without breaking the ABI. It never introduces new functionality.
49 
50 The ABI version (RELEASE_VERSION) is the binary interface, that is, the
51 shared library interface. It has to be increased every time the library
52 changes, e.g., by introducing new functions or changing the memory
53 layout of a class. A patch release must be ABI compatible. A minor
54 release may be ABI compatible -- Test it by replacing the .so for an
55 existing application! A major release necessarily breaks the ABI.
56 
57 LAST_RELEASE is used by a few Common targets, e.g., tarball.

93 ## Hello Library
94 
95 Naming consistency is very important for the automation. Use the
96 lower-case project name for the directory and for the project namespace,
97 e.g., "hello". This is important for include names inside and outside
98 the project. Use the capitalized project name as the library name. This
99 is important for the generated PackageConfig, used by downstream
100 packages to "find_package(Hello)". To configure the library build, the
101 hello CMakeLists sets all variables picked up by the common
102 [common_library](https://github.com/Eyescale/CMake/blob/master/CommonLibrary.cmake),
103 which builds and installs the library and public headers to
104 include/name:
105 
106 @include hello/CMakeLists.txt
107 
108 Each library needs to have the following files, which are consequently
109 created automatically by Common.cmake:
110 
111 * version.h: exports the project API and ABI version for downstream
112  projects. Common.cmake also creates the corresponding version.cpp
113 * defines.h: Project compile-time definitions, i.e., which optional
114  dependencies are available. Includes the actual os-specific defines
115  file generated by FindPackages.cmake.
116 * hello.h: A meta include including all public headers, generated by
117  common_library
118 
119 All generated files are appended to the COMMON_INCLUDES and
120 COMMON_SOURCES cmake variables.
