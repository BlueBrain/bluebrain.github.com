1 Synapse support in Brain {#brain_synapses}
2 ============
3 
4 The low-level data access for synapses provided by Brion needs a complementary
5 API in Brain to support various common higher-level uses cases, like semantic
6 synapse object access or full-circuit synapse density voxelization, to name the
7 two extremes.
8 
9 ## Requirements
10 
11 * Semantic synapse objects that give read-access to all properties
12 * Array access which allows vectorization
13 * Filtering of synapses for pathway use case (projected synapses of two targets)
14 * Lazy-loading to support out-of-core scenarios
15 * User-transparent KVS caching
16 
17 ## New dependencies
18 
19 None

21 ## API
22 
23 ```cpp
24 
25 namespace brain
26 {
27  /*
28  * Synapse GID.
29  *
30  * The GID of a synapse is the a tuple of two numbers:
31  * - The GID of the post-synaptic cell.
32  * - The index of the synapse in the array of afferent contacts
33  * of the post-synaptic cell before pruning/filtering.
34  * GIDs are invariant regardless of how the structural touches are
35  * converted into functional synapses during circuit building.
36  */
37  typedef std::pair< uint32_t, size > SynapseGID;
38 
39  // loading of data during SynapseStream.read(), otherwise load happens on
40  // first touch
41  enum SynapsesPrefetch
42  {
43  SP_NONE = 0, // only loads pre- and post GIDs
44  SP_ATTRIBUTES = 1 << 0, // topological information (section, segment,
45  // distance) and electrical attributes
46  SP_POSITIONS = 1 << 1, // pre/post surface/center positions
47  SP_ALL = SP_ATTRIBUTES | SP_POSITIONS
48  }
49 }
50 
51 class Circuit
52 {
53  // max count for read is gids.size()
54  SynapsesStream getAfferentSynapses( const GIDSet& gids,
55  SynapsesPrefetch prefetch = SP_ALL ) const;
56 
57  // max count for read is gids.size()
58  SynapsesStream getEfferentSynapses( const GIDSet& gids,
59  SynapsesPrefetch prefetch = SP_ALL ) const;
60 
61  // max count for read is min(preGIDs.size(), postGIDs.size())
62  SynapsesStream getProjectedSynapses( const GIDSet& preGIDs,
63  const GIDSet& postGIDs,
64  SynapsesPrefetch prefetch = SP_ALL ) const;
65 };

142 // sequentially & forwarding iteration through synapses for any number of GIDs
143 // not thread-safe, but reentrant
144 class SynapsesStream
145 {
146  bool eos() const;
147 
148  // Remaining count reads before eos()
149  // equals to max - current, max determined by get..Synapses() in
150  // brain::Circuit
151  size_t getRemaining() const;
152 
153  // - async load of synapses for count cells
154  // - no order guarantee
155  // - resulting Synapses can be empty
156  // - count will be clamped if count > getRemaining()
157  std::future< Synapses > read( size_t count = 1 ) const;
158 };

166 ## Examples
167 
168 ### Stream synapse positions for pathway density voxelization
169 
170 ```cpp
171 const brain::Circuit circuit;
172 const brain::SynapsesStream& stream =
173  circuit.getProjectedSynapses( circuit.getGIDs( "mtypetarget_1" ),
174  circuit.getGIDs( "mtypetarget_2" ));
175 std::future< brain::Synapses > future = stream.read();
176 while( !stream.eos( ))
177 {
178  const brain::Synapses synapses = future.get();
179  future = stream.read(); // fetch next
180 
181  const float* __restrict__ posx = synapses.preSurfacePositionX();
182  const float* __restrict__ posy = synapses.preSurfacePositionY();
183  const float* __restrict__ posz = synapses.preSurfacePositionZ();
184  for( size_t i = 0; i < synapses.size(); ++i )
185  doVoxelization( posx[i], posy[i], posz[i] );
186 }
187 ```

211 ## Implementation
212 
213 * Data owned by container in the form of the arrays (transformed from
214  SynapseMatrix), loaded upon construction wrt prefetch, lazy on first touch
215  except for pre- and postGID for connectivity queries and to fulfill the
216  contract for size().
217 * Synapse object stores index to get data from (parenting) container, triggers
218  lazy loading on first touch.
219 * SynapsesStream::read() uses async read of synapse data of the given fraction
220  of GIDs.
221 * OPT: getProjectedSynapses() loads the synapses from the smaller GIDSet and
222  uses the correct side (afferent/efferent) accordingly.
223 
224 ## Issues
225 
226 ### 1: Why does the synapses container has to load on construction?
227 
228 _Resolved: Yes_
229 
230 For an easier implemenation and a consistent API. Determining the size of the
231 container already requires 'touching' all synapse datasets in the current file
232 storage, hence loading attributes or positions in the same go does not waste
233 more time. Regarding memory concerns, the stream approach helps there to only
234 load synapses for a fraction of gids of interest.
235 
236 ### 2: Why there are no set operators like in the BBPSDK synapse container?
237 
238 _Resolved: Yes_
239 
240 Synapses::getProjectedSynapses() was the missing part in the SDK to render most
241 of the set operations useless. It is a more optimized way of filtering.
