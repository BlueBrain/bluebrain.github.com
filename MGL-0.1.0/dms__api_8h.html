Detailed Description

User API for Distributed MultiScale graphs.

This file contains the external API for working with distributed multiscale graphs as supported by the ctools library. It does not include any internals of ctools, so provides a clean, self-contained header file interface for distributed multiscale graphs. Users should only look at the contents of this file (and the documentation generated from it) to use distributed multiscale graphs.

Graph Concept

The graph implementation in ctools represents what graph theory would classify as a bidirectional multigraph. The vertices and edges are first-class objects, and are represented in the API as pointers to opaque structures. Edges are bidirectional in the sense that both vertices can be accessed given an edge reference, and each vertex can access lists of its incoming and outgoing edges. Edges are asymmetric in that there is a clearly defined directionality for edges, and a separation between incoming and outgoing edge lists.

Distributed Graph Concept

Such a graph may be distributed among multiple processes, or parts, such that every vertex in the graph exists on exactly one part. The additional complication of distributing a graph is representing the edges between two parts. These edges are cut in half, and represented by two objects: one half edge on each part. Pairs of half edges point to each other, which means any half edge knows the part number and on-part address of its other half.

Forest Concept

The forest as implemented in ctools is a collection of objects called branches. each branch may have any number of children, which are also branches. Branch A is a parent of B iff B is a child of A. Every branch has zero or one parent branches. A root is a branch that has no parent. When working with a forest, one can access the roots of the forest as one list, and then access the rest of the branches by recursively traversing lists of children.

Distributed Forest Concept

Like graphs, forests may be distributed among multiple parts (processes). Each branch belongs on one part, and parent/child links between different parts are the only complication to manage. An orphan is a branch whose parent exists on a different part. Since orphans cannot be accessed from their parents on the same part, they are stored in a separate orphan list. Each part of a distributed forest has a list of roots and a list of orphans. All the other branches on that part can be reached by traversing down from the roots and the orphans (orphans may still have children on the same part). When a parent/child link is broken between parts, one part ends up with an orphan and the part that has the parent also gets a half-child object. These half-children are placeholders for children on different parts. From any given branch, users may access lists of children and half-children. Each half-child points to the real child branch, which is an orphan somewhere on another part. Similarly, each orphan branch points to its matching half-child object.

Multilevel Graph Concept

A Multilevel Graph is a combination of the Graph and Forest concepts. First, notice that a forest may be separated into levels according to the depth from the roots. A branch is in level N if it takes N moves up the tree to reach the roots. Now, we treat each level as a Graph. That means that branches become vertices as well, and edges may connect them together. The purpose of this complex structure is to represent several models of the same thing, each being more detailed than the last. An object in a simple model is decomposed into many objects in a more complex model, thus the tree structure. Objects in the same model may be arbitrarily associated with one another, thus the graph structure in a level. These objects are still called vertices, as in the graph, but each one is now a combination of a graph vertex and a forest branch.

Distributed Multilevel Graph Concept

Since a Multilevel Graph is a combination of a graph and a forest, and both graphs and forests can be distributed as described above, then the whole multilevel graph can also be distributed. Each vertex of a multilevel graph belongs on one part, edges between parts are handled as described in the Distributed Graph Concept, and parent/child relationships between parts are handled as described in the Distributed Forest Concept. Since the forest provides a good iteration mechanism, this is the way multilevel graph vertices are accessed: by root, orphan, and child lists.
