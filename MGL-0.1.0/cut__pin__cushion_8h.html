Detailed Description

the cut version of pins and cushions, introduces needles

the pin and cushion structures model a many-to-one relationship. This file makes use of the cut structure to define a representation for a distributed one-to-many relationship. Essentially, the objects can be distributed across processes and the relation may span between two processes. We use the analogy of a pin being cut in two. If one looks at a pin and a needle, one can understand that a pin is a needle with a small bead of plastic on top, called the pin head. Here, we say that when a pin is cut in half, one half is a needle and the other half is a pin_head. By using the cut structure, a needle and a pin_head remain linked together by global pointers. next, we extend the cushion structure by adding a list for needles. in this way, both pins and needles may be pushed into a cushion. To convert a local one-to-many relationship modeled by one cushion and many pin objects, one simply has to cut all the pins that are on a different part from the cushion. The result of this cut is needle which remains pushed into the cushion on the same part, and maintains a global pointer to the pin_head, which is on a different part. The pin_head, in turn, maintains a global pointer to the needle.

In addition, we can use the generic nature of the cut structure to allow two needles to point to each other, which enables nets to be parallelized by cutting ropes. We also provide a container for pin heads, which stores a group of pin_head objects, since the cushion, being on a different part, can no longer be used as a container for them. This is used for storing orphans in a cut forest.
