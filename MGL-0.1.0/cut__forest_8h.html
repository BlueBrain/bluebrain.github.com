Detailed Description

a cut version of forest, based on cut_tree

This structure and functions create a cut version of the forest data structure. The cut version requires a special list for orphan branches. This adds a few extra functions to the API. There are also functions for making and gluing orphans.

void forest_remove_orphan ( struct cut_forest *  f,
struct cut_branch *  b 
)

Remove an orphan branch.

Remarks:
due to the "child adoption" behavior of the normal forest removal procedure, children of b must be moved up. If b is an orphan, however, we cannot just put the children in b's sibling list because that list is only for orphans, and the children by definition are not orphans. So, we need a pointer to the forest for access to the ophan list (orphan_soil).
Parameters:
fthe forest
bthe branch to remove

Glues an orphan and a cut_child into a child branch.

This function takes cut_branch b and cut_child c in the same forest and replaces c with b. the two are expected to match, thus b was an orphan and becomes a child.

Parameters:
ba orphan branch to convert to a child
cthe cut child structure matching b

Makes a branch into an orphan.

This function cuts the parent/child relationship between b and its parent. As a child, b is replaced by the cut child c. b is then converted into an orphan branch and placed in f's orphan list.
