Detailed Description

binary trees

These data structures and functions implement binary trees with parent pointers, and include several complex operations on binary trees. Currently, these are helpers for the AVL tree implementation, but are reusable as a generic binary tree implementation.

typedef enum bi_comparison(* bi_compare_t)(struct bi_node *a, struct bi_node *b)

binary tree node comparison function type

This typedef describes the signature of the callback functions used to compare two binary tree nodes. Such a function should take two nodes as arguments and return a value of the bi_comparison enum. returning bi_less means a < b, and so on.

Get the deepest binary node on the given side.

This function starts at parent and repeatedly moves in direction side. It returns the last node encountered before NULL. If side is bi_left, it returns the leftmost node in a subtree rooted at parent. If parent has no left children, it will return the parent.

Get the first ancestor from a given side.

This function travels up the tree starting at child, and returns the first node for which the traversal went up its child link on the given side. If side is bi_left, the first time the funtion travels from a node's left child to that node, it will return that node. This is also useful during in-order traversal.

struct bi_node* insert_bi_node ( struct bi_tree *  t,
struct bi_node *  n,
bi_compare_t  compare 
) [read]

Insert a node into a binary search tree.

This function assumes the given tree is a correct search tree, and tries to insert the given node according to the given comparison function. If it finds a node in the tree that compares equal to the given node, it will return the node in the tree and not insert the given node. In this way, this function is both the search and insertion function.

Parameters:
tthe tree
nthe node to try to insert
comparethe comparison callback function
Returns:
an equal node in the tree, or n if n is inserted
