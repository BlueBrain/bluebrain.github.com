Detailed Description

AVL trees.

These data structures and functions implement the well-know yet complex AVL self-balancing binary search tree structure.

There are existing implementations of self-balancing trees, and it is worth describing what sets this one appart: There exists an implementation of AVL trees in the c-algorithms library. Their implementation uses much more memory, in part because it relies on void* to store data. Our implementation uses the same optimization as the C++ std::set, which is to require that structures include the avl_node as a member, and uses callback functions to compare nodes. The C++ std::map and std::set are implemented as red-black trees. Their implementation takes charge of the memory allocation, and restricts structures to be members of only one map. A comparison of our AVL tree against std::set shows that it is twice as fast when it can avoid dynamic memory allocation.

struct avl_node* insert_avl_node ( struct avl_tree *  t,
struct avl_node *  n,
bi_compare_t  compare 
) [read]

Insert an AVL node into an AVL tree.

This function tries to insert the given node into the tree using a comparison function. If another node is found in the tree that compares the same as the given node, The node already in the tree will be returned. In this way, this function is both the search and insertion function. As is widely known, this operation has O(log(n)) runtime where n is the number of nodes in the tree.

Parameters:
tthe tree
nthe node
comparethe comparison function
Returns:
the node in the tree equal to n, or n if n has been inserted
