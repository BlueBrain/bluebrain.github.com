107 # Release 1.5.0 (7-Jul-2015)
108 
109 * Add RESTING_VOLTAGE constant.
110 * Add support for binary spike reports.
111 * Add support for spike writer plugin.
112 * Add async IO support in brion::CompartmentReportMap.
113 * brion::CompartmentReportBinary::updateMapping() more robust.
114 * brion::SpikeReport API extended to support stream-based reports. A reference
115  implementation of a stream-based spike report plugin is also provided.
116 * Ensure thread-safety with non-threadsafe HDF5 library.
117 * Morphology class implementation changed to support the Lunchbox plugin
118  mechanism.
119 * New morphology plugin to read SWC morphologies.
120 * [#3](https://github.com/BlueBrain/Brion/pull/3):
121  New morphology converter tool.
122 * New null compartment report for benchmarks.
123 * New spike converter tool.
124 * Optimized function Circuit::get for large circuits (cut down the loading time
125  of the 3.1 M neuron circuit by 6.5 seconds in a Core i7-4930K @ 3.40GHz).
126 * Replace use of iostreams by lunchbox::MemoryMap.
127 * Several bugfixes and cleanups.
128 
129 # Release 1.4.0 (8-Oct-2014)
130 
131 * Old report readers have been adapted to be static plugins using
132  lunchbox::IOPluginFactory.
133 * New spike report reader plugin for NEST spike reports. The plugin can
134  parse multiple report file using shell wildcards (*, ?) in the
135  filepath provided.
136 * New constructor for brion::CompartmentReport accepting an URI and the desired
137  access mode deprecates the old read and write constructors.
138 * New compartment report based on key-value stores supported by
139  Lunchbox::PersistentMap.
140 * Spike loading has been optimized (in a 12-core GPU the expected
141  speed-up is above 4x for large files).
