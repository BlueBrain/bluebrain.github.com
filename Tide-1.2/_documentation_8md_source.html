7 ## Directory Layout
8 
9 * [CMake](https://github.com/Eyescale/CMake#readme): subdirectory
10  included using git externals. See below for details.
11 * tide: Contains the main libraries of the project:
12  * core: The core library contains utilities, helpers and shared classes that
13  are used by the master and wall applications.
14  * master: The master library contains the control GUI and other
15  application-control logics.
16  * wall: The wall library contains the QML2 rendering engine for the supported
17  content types and their synchronization between other wall processes.
18 * apps: Applications delivered with the project:
19  * tide: The main application which launches the master and wall processes
20  according to the provided configuration file.
21  * TideMaster: The master application launched by tide which provides a control
22  user interface to open contents and sessions, change options etc.
23  * TideWall: The wall application launched for each wall segment launched by
24  tide which presents all opened contents.
25  * TideForker: An auxilary process launched by tide which forks new processes
26  like LocalStreamer.
27  * Launcher: The application which streams the Qml control panel for browsing
28  documents, sessions and launching applications.
29  * LocalStreamer: An application that generates content according to launch
30  parameters and streams that content to the local tide
31  instance. The webbrowser content for instance is one
32  exisiting implementation.
33  * pyramidmaker: A script that generates a TIFF image pyramid from a big
34  source image using ImageMagick's convert tool. Image pyramids
35  can be loaded and rendered by Tide more efficently.
36 
37 * tests: Unit tests.
38 * doc: Doxygen and other documentation.
39 * examples: Example xml configuration files, installed under share/Tide.

41 ## CMakeLists
42 
43 The top-level CMakeLists is relatively simple due to the delegation of
44 details into the CMake external. It starts with the project setup which
45 defines the project name and includes the CMake/common git external.

47 ## CMake
48 
49 All BBP projects rely on a common
50 [CMake repository](https://github.com/Eyescale/CMake) which provides
51 sensible defaults for compilation, documentation and packaging. It is
52 integrated as a CMake/common subtree as described in the
53 [Readme](https://github.com/Eyescale/CMake#readme).
