3 ![](doc/BBPLOGO350.jpg)

5 Welcome to Brion, a C++ project for read and write access to Blue Brain data
6 structures, including BlueConfig/CircuitConfig, Circuit, CompartmentReport,
7 Mesh, Morphology, Synapse and Target files.
8 
9 Brion can be retrieved by cloning the
10 [source code](https://github.com/BlueBrain/Brion.git).
11 The [latest API documentation]
12 (http://bluebrain.github.io/Brion-1.9/index.html) can be found on
13 [bluebrain.github.io](http://bluebrain.github.io).
14 Additional documentation exists for the [Python wrapping of Brain]
15 (python/index.html).
16 
17 To keep track of the changes between releases check the [changelog](@ref Changelog).

19 ## Features
20 
21 Brion provides two libraries Brion and Brain. The former is a collection of file
22 readers and writers intended for low level access to the data model. The latter
23 is a set of higher level classes that wrap low level data objects with a
24 use-case oriented API.
25 
26 ### IO library
27 
28 This is the core library provided by Brion. It includes classes for reading
29 and writing files which store the Blue Brain data model.
30 
31 * Fast and low-overhead read access to:
32  * Blue configs (brion::BlueConfig)
33  * Circuit description (brion::Circuit)
34  * H5 Synapses data (brion::SynapseSummary, brion::Synapse)
35  * Target (brion::Target)
36  * BBP binary meshes (brion::Mesh)
37  * BBP H5 morphologies and SWC morphologies (brion::Morphology)
38  * Compartment reports (brion::CompartmentReport)
39  * Spike reports (brion::SpikeReport)
40 * Fast and low-overhead write access to:
41  * Compartment reports (brion::CompartmentReport)
42  * BBP binary meshes (brion::Mesh)
43  * BBP H5 morphologies (brion::Morphology)
44  * Spike reports (brion::SpikeReport)
45 * Basic [data types](@ref brion/types.h) to work with the loaded data using
46  [Boost](http://www.boost.org/doc/libs),
47  [Lunchbox](http://eyescale.github.io/Lunchbox-1.14/index.html),
48  [vmmlib](http://eyescale.github.io/vmmlib-1.12/index.html).
49 
50 ### High level library
51 
52 The higher level library is called Brain and it provides:
53 
54 * brain::Circuit to facilitate loading information about cells, morphologies (in
55  local and global circuit coordinates) and synapses.
56 * brain::neuron::Morphology with higher level functions to deal with
57  morphologies.
58 * brain::Synapses and brain::Synapse for array and object access to synapses.
59 
60 ## Building
61 
62 Brion is a cross-platform library, designed to run on any modern operating
63 system, including all Unix variants. Brion uses CMake to create a
64 platform-specific build environment. The following platforms and build
65 environments are tested:
66 
67 * Linux: Ubuntu 16.04, RHEL 6.8 (Makefile, Ninja)
68 
69 Building from source is as simple as:
70 
71  git clone --recursive https://github.com/BlueBrain/Brion.git
72  mkdir Brion/build
73  cd Brion/build
74  cmake -GNinja -DCLONE_SUBPROJECTS=ON ..
75  ninja
