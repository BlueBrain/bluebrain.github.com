compartmentConverter
Usage: compartmentConverter input-uri [output-uri=dummy://] [options]
Supported input and output URIs:
Blue Brain binary compartment reports: [file://]/path/to/report.(bin|rep|bbp)
Benchmark drain (write): dummy://
Benchmark dummy source (read): dummy://[?size=size-in-MB][&randomValues]
default 1024MB of dummy report data
SONATA HDF5 compartment reports: [file://]/path/to/report.(h5|hdf5)[?[cache_size=(auto|num_bytes)&][cells_to_frames=(inf|ratio)&][chunk_size=bytes]]
Byte counts can by suffixed by K or M.
The cache is disabled by default, auto will reserve space fora whole frame or trace, whatever is bigger. The actual size depends on the chunk dimensions. For files with row or column layouts the auto cache size is limited to 1GB.
Blue Brain HDF5 compartment reports: [file://]/path/to/report.(h5|hdf5)
Options:
-h [ --help ] Produce help message
-v [ --version ] Show program name/version banner and exit
-e [ --erase ] arg Erase the given report (map-based reports only)
-m [ --maxFrames ] arg Convert at most the given number of frames
-g [ --gids ] arg List of whitespace separated GIDs to convert
-c [ --compare ] Compare written report with input
-d [ --dump ] Dump input report information (no output conversion)
