8 ## Livre 0.3
9 
10 * The hierarchical information for the Dash tree is now implicit in each node ID,
11  effectively becoming a flat structure. This change has a positive impact on the
12  performance and memory use
13 * Add frustum culling
14 * Synchronous mode (wait until every block has been processed before rendering them)
15  is now available as a command line option
16 * Add time support, implementing a new animation mode that triggers a new frame
17  after the rendering of the previous one is finished
18 * Add the possibility of specifying the rage of frames to render from the command line
19 * Add livre_batch.py script for offline image series rendering
20 
21 * Command line parameters general cleanup
22 * Add command line parameters for RESTBridge (hostname, port, ZeroEQ schema)
23 * Add command line option to enable ZeroEQ camera synchronization
24 * Add command line parameters to specify the minimum and maximum levels of detail to render
25 * Add command line parameter to set the camera lookAt vector to define its orientation
26 
27 * Fix a problem that made the texture cache not to unload unused elements
28 * Fix the LOD computation based on the screen height
29 * Graceful exit when ZeroEQ EXIT event is received
30 * Several fixes in the tree creation and the LOD-selection algorithm
31 * Fix the "Empty servus implementation" exception that was raised when zeroconf wasn not available
32 * Do not setup ZeroEQ subscribers for REST if not requested
33 
34 * Refactor the code to setup the data and texture UploadProcessors, removing race
35  conditions and fixing crashes (at exit and with multiple local GPUs) in the process
36 * Window title has been improved to show the application name and its version number
37 * Frames can now be written to disk as PNG files
38 * Qt5 replaces Qt4 dependency (used for the GUI in the transfer function editor)
39 * General code cleanups and improvements
