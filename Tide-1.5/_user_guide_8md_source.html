4 This document is a short how-to to start running Tide.

6 # Usage
7 
8 Run 'tide' from the bin folder to launch the application.
9 
10 Some useful command line options:
11 
12 * \-\-config can be used to launch the application with a specific
13  configuration. A default test configuration is used if none is provided.
14 * \-\-help lists all the available options.
15 
16 ## %Configuration file
17 
18 A single json configuration file is required to launch the application. It
19 defines the number of processes to launch, the hosts on which they have to run
20 and the list of windows that each one of them will display.
21 
22 \include examples/configuration_1x3.json
23 
24 More examples can be found in the examples folder of the source directory, or
25 installed under ${install_prefix}/share/Tide/examples.
26 
27 ## TIFF image pyramids
28 
29 Tide can open any image pyramid saved in standard TIFF file format. To convert
30 an existing image to a TIFF pyramid one can use for instance ImageMagick:
31 > convert myimage.xyz -monitor -define tiff:tile-geometry=512x512 -compress jpeg 'ptif:myimage.tif'
32 For more convenience, use the *pyramidify* tool provided by Tide:
33 > pyramidify myimagefolder/*.png
34 Which is equivalent to calling manually the *pyramidmaker* script (also provided
35 by Tide) for each image:
36 > pyramidmaker myimage.xyz myimage.tif
37 
38 ## Stereo 3D movies
39 
40 Tide can render 3D movies in side-by-side format if the movie file contains
41 information about the stereo mode. For example, the webm and mkv containers can
42 store such metadata. A valid side-by-side stereo movie can be created using
43 [ffmpeg](https://ffmpeg.org/ffmpeg-formats.html#Metadata):
44 
45 > ffmpeg -i left_right_clip.mpg -an -c:v libvpx -metadata stereo_mode=left_right -y stereo_clip.webm
46 
47 Just changing the container for mkv (no re-encoding):
48 
49 > ffmpeg -i left_right_clip.mp4 -an -c:v copy -metadata stereo_mode=left_right -y left_right_clip.mkv
50 
51 To check if a movie has the correct stereo metadata, use ffprobe:
52 
53 > ffprobe stereo_clip.webm
54 Metadata:
55 stereo_mode : left_right
56 Side data:
57 stereo3d: side by side

59 ## Open ports
60 
61 Tide listens on the following ports:
62 
63 * TCP port 1701 - incoming Deflect connections.
64 * UDP port 3333 - TUIO messages (if using TuioTouch Qt plugin).
65 * TCP port 8888 - REST interface (if compiled with ZeroEQ support),
66  configurable.
67 
68 ## OSX notes
69 
70 The following steps might be required to run the application on OSX.
71 
72 * Allow 'Remote Login' in 'System Preferences' -> Sharing
73 * Allow passwordless login on the same machine:
74  ~~~~~~~~~~~~~{.sh}
75  cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys
76  ~~~~~~~~~~~~~
77 * Reference your hostname explicitly in /etc/hosts to point to 127.0.0.1.
78  Example:
79  ~~~~~~~~~~~~~{.sh}
80  hostname> bluebrain077.epfl.ch
81  # Add to /etc/hosts:
82  127.0.0.1 bluebrain077.epfl.ch
83  ~~~~~~~~~~~~~
84 
85 Multiple windows are not supported, only the basic single-window configuation
86 works:
87 > bin/tide --config $PWD/../examples/configuration_1x1.json
