Detailed Description

Handles window creation, association and updates for pixel streamers, both local and external.

The association is one streamer to one window.

void PixelStreamWindowManager::requestFirstFrame ( QString  uri)
signal

Emitted when handleStreamStart is called for a stream which already has a window, such as the Launcher or a Webbrowser.

For external streamers, the requestFrame signal comes from the Wall processes when the window has been opened. For local streamers, however, the window is opened before the deflect::Stream is started so the deflect::server::FrameDispatcher discards it and no frames would be displayed otherwise.
