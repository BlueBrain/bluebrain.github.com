4 # Release 1.5 (git master)
5 
6 * [243](https://github.com/BlueBrain/Tide/pull/243):
7  The Launcher has an improved file browser:
8  - case-insensitive sorting.
9  - list view: sorting by name, size or modification date.
10  - scroll bar for faster navigation.
11  - tap-and-hold on a folder to open all the files in it.
12 * [241](https://github.com/BlueBrain/Tide/pull/241):
13  New option to display the file paths in the window title bars.
14  Web interface:
15  - New "contents" dialog to help finding the location of open files.
16  - Small bugfixes.
17 * [239](https://github.com/BlueBrain/Tide/pull/239):
18  The behavior of the 1:1 button changed for contents that would exceed
19  the surface dimensions in 1:1 size. Previously such windows were hard
20  to minimize / close again. Being in fullscreen (with the desired 1:1 size)
21  gives an easy way to exit and restore the window to a minimized size.
22 * [238](https://github.com/BlueBrain/Tide/pull/238):
23  The location for temporary contents uploaded through the web interface is
24  configurable. This is particularly useful in setups with multiple machines,
25  where temporary files must be stored on a shared file system instead of /tmp
26  on the master server for render clients to be able to access them.
27 * [237](https://github.com/BlueBrain/Tide/pull/237):
28  Support for multiple rendering surfaces, each with its own background and set
29  of contents. This enables Tide to drive complex visualization installations
30  composed of separate wall and floor projection systems, for example.
31 * [236](https://github.com/BlueBrain/Tide/pull/236):
32  A new .json configuration file format replaces the old .xml configurations.
33  Those are now deprecated and support for xml will be removed in the future
34  Tide 1.6 release. Use the provided tideConverter command line tool to easily
35  convert existing xml configurations to json format.
36 * [232](https://github.com/BlueBrain/Tide/pull/232):
37  Tide was upgraded to C++14 and requires a compatible compiler
38  (GCC 5, Clang 3.4).
39 * [229](https://github.com/BlueBrain/Tide/pull/229):
40  Different models of Planar TV can be controlled.
41 * [226](https://github.com/BlueBrain/Tide/pull/226):
42  New pyramidify tool to simplify the creation of multiple TIFF image pyramids.
43 * [224](https://github.com/BlueBrain/Tide/pull/224):
44  Number of touch points required to power on the screens is configurable.
45 * [220](https://github.com/BlueBrain/Tide/pull/220):
46  Power control feature available in setups with mutliple controllers.
47 * [214](https://github.com/BlueBrain/Tide/pull/214):
48  Rockets replaces ZeroEQ dependency for the REST interface.
49 * [212](https://github.com/BlueBrain/Tide/pull/212):
50  Removed legacy Webkit webbrowser (Qt < 5.6), require boost 1.56 or later.
51 * [211](https://github.com/BlueBrain/Tide/pull/211):
52  Qt's TuioTouch plugin replaces BlueBrain's TUIO subproject. Note: when not in
53  headless mode, the touch points can only be delivered when the main window's
54  Qml item has active focus.

66  - Replaced screenWidth/Height with displayWidth/Height, which refers
67  to the resolution of an invidual physical display.

72  - mullionWidth/Height -> bezelWidth/Height.
73  - Removed bezelsPerScreenX/Y which became redundant.
74  Fixes in web interface related to bezel and configuration.

76  New features in html interface:

82  Support for textures / PixelStreams in OpenGL (bottom-up) row order.
83 * [179](https://github.com/BlueBrain/Tide/issues/179):
84  Allow turning pixel stream compression on/off while streaming (bugfix).

86  Hardware swap synchronization for NVidia Quadro sync cards.

88  Fix crash with malformed encoded pixel streams, e.g. JPEG quality > 100
89 * [169](https://github.com/BlueBrain/Tide/issues/169):
90  Performance improvement for PixelStreams with small tile size (64x64).
91 * [167](https://github.com/BlueBrain/Tide/pull/167):
92  Tide can be locked to prevent unwanted streams from opening or actions from
93  the HTML interface, e.g. during a presentation.
94 * [162](https://github.com/BlueBrain/Tide/issues/162):
95  Movies and pixel streams can be rendered at 60 fps (up from 30 fps).
96 * [161](https://github.com/BlueBrain/Tide/issues/161):
97  Improved automatic layout for focused windows, no longer in one line.
98 * [151](https://github.com/BlueBrain/Tide/issues/151):
99  Added support for playing stereo 3D movies.
100 * [147](https://github.com/BlueBrain/Tide/issues/147):
101  Performance improvement: added support for multiple windows per process.
102  The windows can be located on different X11 displays and still share resources
103  such as decoded images and movie frames.
104  Example gains:
105  - On a machine with 3-GPUs, CPU load is reduced by 66% when decoding a
106  fullscreen movie.
107  - Using passive stereo windows, an (additional) 50% CPU load is saved when
108  decoding movies.
