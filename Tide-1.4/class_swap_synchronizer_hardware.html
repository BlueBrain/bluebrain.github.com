Detailed Description

Hardware swap synchonizer using GL extensions (for NVidia Quadro GSync card).

Given the following setup:

Node #1: process A {Left window, Right window} Node #2: process B {Left window, Right window}

Where each window has its own (non-sharing) GLContext and render thread. The startup procedure is: — global barrier — A-L joinSwapGroup(1) | A-R joinSwapGroup(1) || B-L joinSwapGroup(1) | B-R joinSwapGroup(1) — A local barrier — || — B local barrier — A joinSwapBarrier(1,1) || B joinSwapBarrier(1,1) — global barrier —

And the shutdown procedure is: — global barrier — A leaveSwapBarrier() || B leaveSwapBarrier() A-L leaveSwapGroup() | A-R leaveSwapGroup() || B-L leaveSwapGroup() | B-R leaveSwapGroup()

The {join|leave}SwapBarrier must be executed by a single GL context per machine.
