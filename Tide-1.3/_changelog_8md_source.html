17  - the web interface no longer blocks the application

51 # Release 1.2.1 (16-12-2016)
52 
53 * [113](https://github.com/BlueBrain/Tide/pull/113):
54  Fixed several issues linked to ZeroEQ with the 1.2 release:
55  - The REST interface was constantly polling, causing a 100% idle CPU load.
56  - This caused troubles with window animations in headless mode, resulting in
57  touch events hitting the background or some other window.
58  - cmake -DINSTALL_PACKAGES=1 did not install cppnetlib's dependencies.
59  Additionnally, the REST "open" command can be given a directory path to open
60  all the contents inside it. This feature was previously inaccessible in
61  headless mode.

63 # Release 1.2 (09-12-2016)
64 
65 * [111](https://github.com/BlueBrain/Tide/pull/111):
66  Multiple bugfixes:
67  - Fixed a bug that could cause focused windows to overlap.
68  - Corrected the global thread pool size in multi-node configurations.
69  - The application no longer blocks on startup if an error occurs, such as the
70  chosen REST port being already in use.
71 * [109](https://github.com/BlueBrain/Tide/pull/109):
72  New REST command to capture full resolution screenshot of the display wall.

74  Tide can now be started in "headless" mode, without a visible control window.
75  This simplifies the deployment on installations where no dedicated control
76  monitor is available, where previously the master window had to be run inside
77  a VNC server.
78 * [104](https://github.com/BlueBrain/Tide/pull/104):
79  Tide can be controlled from the python BBP viztools.

81  The Launcher can generate thumbnails much faster (requires Qt 5.6.3 or 5.7.1).
82 * [97](https://github.com/BlueBrain/Tide/pull/97):
83  Webbrowsers can be saved and restored from sessions and display the page title
84  in their title bar.
85 * [96](https://github.com/BlueBrain/Tide/pull/96):
86  Bug fixes in Whiteboard application.
87 * [95](https://github.com/BlueBrain/Tide/pull/95):
88  More consistent and intuitive user experience:
89  - Double-tap a window to make it fullscreen
90  - To present multiple contents side by side, select them on the Desktop with a
91  single tap then press any of their "eye" icons.
92  - Tap the background to exit any presentation mode.
93  - On the Desktop, tap the background to unselect all windows.
94 * [93](https://github.com/BlueBrain/Tide/pull/93):
95  Added a whiteboard application with support for:
96  - Drawing in multiple colors and with different brush sizes
97  - Saving an image to a png file
98  - Extending/shrinking a canvas on the size change of applications window
99 * [92](https://github.com/BlueBrain/Tide/pull/92):
100  Improvements to the WebEngine webbrowser (Qt 5.6 or later):
101  - Plugins enabled (flash)
102  - Replace "select" system drop-down lists with html equivalents (as was
103  already done for the Webkit webbrowser)
104 * [90](https://github.com/BlueBrain/Tide/pull/90):
105  The fullscreen mode has some new features:
106  - Users can now enlarge and pan contents (without the need to use the zoom)
107  - A double-tap toggles between *adjusted* and *maximized* fullscreen
108  coordinates
109  - Movies can be played/paused from the side control bar
110  - The keyboard can be opened from the side control bar for streamed contents
111  Also, the keyboard icon is no longer shown for the new WebEngine-based
112  webbrowsers who use their own integrated keyboard.
113 * [85](https://github.com/BlueBrain/Tide/pull/85):
114  Fix issues affecting Qt 5.7 caused by a conflict with Tide's virtual keyboard.
115 * [84](https://github.com/BlueBrain/Tide/pull/84):
116  Multitouch improvements [DISCL-383]:
117  - Tap and DoubleTap gestures work with any number of fingers
118  - DoubleTap a window with two fingers to make it fullscreen
119  - The changes also fixed bug [80](https://github.com/BlueBrain/Tide/issues/80)
120  on Qt >= 5.6, which could lead to an obscure race condition in the QV4
121  JavaScript engine when changing the State property of the ContentWindow
122  between resizing / moving / none.
123  Deflect adaptations [DISCL-386]:
124  - Clients now receive PINCH events instead of WHEEL events
125  - Clients also receive raw touch events (touch point added, updated, removed)
126 * [83](https://github.com/BlueBrain/Tide/pull/83):
127  Added TIDE_IGNORE_MPI_THREADSAFETY CMake option for Ubuntu 14.04 after #82
