1 Building Tide {#building}
2 ============
3 
4 Tide uses CMake 3.1 or later to create a platform-specific build environment.
5 
6 ## Building from Source
7 
8 The standard build procedure using git and CMake is the following:
9 
10  git clone https://github.com/BlueBrain/Tide.git
11  mkdir Tide/build
12  cd Tide/build
13  cmake ..
14  make
15 
16 or using the Ninja code generator (recommended):
17 
18  cmake -GNinja ..
19  ninja
20 
21 ## External Dependencies
22 
23 Tide builds on top of standard and well tested libraries for core
24 functionalities such as rendering, serialization, file handling and network
25 communication. Those dependencies are:
26 * Boost 1.54 or later
27 * MPI with MPI_THREAD_MULTIPLE support (openmpi 1.6.5 or later recommended)
28 * Qt 5.4 or later (5.7.1 or later recommended)
29 
30 In addition, it also depends on some external projects that are automatically
31 cloned by CMake during the configure step. They come with their own additional
32 requirements:
33 * Deflect: streaming of contents and applications
34  - libjpeg-turbo
35 * TUIO: multitouch interface
36 * VirtualKeyboard: on-screen virtual keyboard (Qml)
37 * ZeroEQ: http / REST interface (technically optional, but still needed to
38  operate on-screen Launcher panel)
39  - Pthreads
40  - ZeroMQ
41 
42 Some additional libraries are searched by CMake to enable useful extra
43 features if they are found:
44 * Cairo + RSVG (>= 2.36.2): optimal %SVG support (QtSVG used as a fallback)
45 * Cairo + poppler-glib: optimal PDF support
46 * poppler-qt5: fallback PDF support
47 * FFMPEG: playing movies
48 * TIFF: display large images (TIFF image pyramids)
49 
50 ### Automatic Package Installation
51 
52 To ease the process of searching and installing all the necessary system
53 packages, an automatic installation process is provided. It is triggered by
54 passing a specific argument to CMake during the configuration step.
55 Currently, only Ubuntu (via apt-get) and OSX (via macports) are supported.
56 
57 To run the automatic install, do the following:
58 
59  cd Tide/build
60  cmake .. -DINSTALL_PACKAGES=1
61 
62 One could also look directly at the TIDE_DEB_DEPENDS and TIDE_PORT_DEPENDS
63 entries in the top-level CMakeLists.txt of each (sub)project and install the
64 packages manually.
65 
66 ## Supported Platforms
67 
68 This section gives more detailed information on building the software on some
69 popular platforms.
70 
71 Unfortunately, many Linux distributions lack at least one of the required
72 components which must be installed manually. New users are advised to consider
73 Ubuntu 16.04 which is one of the simplest option overall.
74 
75 ### Ubuntu 16.04
76 
77 Tide works almost out of the box on Ubuntu 16.04. The only issue is that the
78 openmpi / mpich packages are either buggy or lack support for
79 MPI_THREAD_MULTIPLE.
80 
81 To boostrap the installation of Tide a on fresh install, do:
82 
83  sudo apt install git cmake
84  git clone https://github.com/BlueBrain/Tide.git
85  mkdir Tide/build
86  cd Tide/build
87  cmake .. -DINSTALL_PACKAGES=1
88 
89 After this step all the necessary system packages should be installed. Then use
90 the following receipe to build and use a proper openmpi replacement:
91 
92  VERSION=1.10.3
93 
94  OPENMPI_MIRROR=http://www.open-mpi.org/software/ompi/v1.10/downloads
95  OPENMPI_FOLDER=openmpi-${VERSION}
96  OPENMPI_TARBALL=${OPENMPI_FOLDER}.tar.gz
97 
98  wget ${OPENMPI_MIRROR}/${OPENMPI_TARBALL}
99  tar xzf ${OPENMPI_TARBALL}
100  cd ${OPENMPI_FOLDER}
101 
102  ./configure --enable-mpi-thread-multiple --prefix=$HOME/opt/openmpi
103  make install
104 
105 And export the corresponding environment variables (add in ~/.bashrc):
106 
107  OPENMPI_ROOT=$HOME/opt/openmpi
108  export PATH=$OPENMPI_ROOT/bin:$PATH
109  export LD_LIBRARY_PATH=$OPENMPI_ROOT/lib:$LD_LIBRARY_PATH
110 
111 Note: older openmpi versions have been tested to work as well (1.8.7, 1.6.5).
112 
113 Check that the correct openmpi version is in the PATH:
114 > which mpiexec
115 and clear the CMakeCache (or just start with a fresh build folder) to ensure
116 that CMake does not use the incompatible system openmpi.
117 
118 Warning: It seems that OpenMPI does not build correctly if OpenCL is installed.
119 The only know solution is to unistall OpenCL, build openmpi then re-install
120 OpenCL.
121 
122 Alternative: If the above does not work for you, try installing the libmpich-dev
123 system package. Tide will then only run in single-window mode (using
124 configuration_1x1.xml).
125 
126 ### Ubuntu 14.04
127 
128 Tide has been extensively developed and tested on Ubuntu 14.04. Only a few
129 system packages are outdated and must be replaced:
130 * CMake 2.8: download recent CMake 3 binaries
131  [from here](https://cmake.org/download/) and export PATH to *cmake* executable
132 * Qt 5.2.1: use Qt 5.4.1 (or later) installer downloaded from
133  [the Qt website](http://download.qt.io) and export (add in ~/.bashrc):
134 
135  QT_HOME=$HOME/Qt5.4.1/5.4/gcc_64
136  export PATH=$QT_HOME/bin:$PATH
137  export LD_LIBRARY_PATH=$QT_HOME/lib:$LD_LIBRARY_PATH
138  export QML_IMPORT_PATH=$QT_HOME/qml
139 
140 * cppcheck + lcov: build custom cppcheck-1.68 and lcov-1.11 (or newer) and
141  export PATH to the binaries (only needed for *tests* and *doxygen* targets)
142 
143 The system libopenmpi-dev package (verion 1.6.5-8) only reports
144 MPI_THREAD_SERIALIZED support. Despite this, it works just the same as if it had
145 full MPI_THREAD_MULTIPLE support. The MPI thread support check can thus be
146 safely disabled when configuring the project on this platform:
147 
148  cmake .. -DTIDE_IGNORE_MPI_THREADSAFETY=ON
149 
150 ### RedHat 6
151 
152 Tide has been built and used in production on RHEL 6.7 machines. However, the
153 deployment requires a large amount of Linux modules to replace the outdated
154 system packages and even then some of the latest features remain unavailable
155 (Cairo backend). This system is therefore not recommended for new deployments.
156 
157 For those interested, here is a list of modules used at BlueBrain at the time of
158 writing (September 2016):
159 * cmake/3.2.3
160 * gcc/4.8.3 (+ gdb/7.5.1)
161 * boost/1.54.0
162 * openmpi/1.8.7
163 * qt/5.4.1
164 * poppler/0.45.0
165 * libzmq/4.0.6
166 * python/3.4.3-1
167 * doxygen/1.8.9.1
168 * cppcheck/1.68
169 * lcov/1.11
170 
171 ### MacOS
172 
173 Tide builds and runs on OSX (x86_64), but has limited features and testing.
174 
175 The required libjpeg-turbo cannot be installed via macports because it conflicts
176 with the regular libjpeg. A separate installer must be obtained from
177 [sourceforge](https://sourceforge.net/projects/libjpeg-turbo/).
178 
179 ### Windows
180 
181 There is no plan to support Windows at the moment.
182 
183 However, since most dependencies are open-source and cross-platform, one might
184 be able to build a Windows version of Tide given sufficient time and effort.
