Detailed Description

Extends the asynchronous receiver by providing cancelable requests, which will provide progress updates during their execution.

The cancel notification needs to have the following payload:

1 {
2  "jsonrpc": "2.0",
3  "method": "cancel",
4  "params": { "id": <request_to_cancel> }
5 }

The progress notification will have the following payload:

1 {
2  "jsonrpc": "2.0",
3  "method": "progress",
4  "params": {
5  "id": <request_id>
6  "amount": float
7  "string": operation
8  }
9 }
